# Security Policy

## Supported Versions

Flint is pre-1.0. Only the latest published version receives security fixes.

| Version | Supported |
| ------- | --------- |
| Latest  | ✅        |
| Older   | ❌        |

## Reporting a Vulnerability

**Please do not open public GitHub issues for security vulnerabilities.**

To report a vulnerability:

1. Use GitHub's [private vulnerability reporting](https://github.com/flintdevv/flint/security/advisories/new) (preferred).
2. Or email: `security@flintspark.dev` *(once domain is configured; until then, use GitHub advisories)*

We aim to acknowledge reports within 72 hours and provide a fix or detailed response within 14 days for confirmed issues.

## Disclosure Policy

We follow responsible disclosure. Once a fix is available and users have had reasonable time to upgrade, we publish a CVE and a public advisory.
